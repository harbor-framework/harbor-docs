---

Harbor aggregates dataset rewards with a metric.

## Built-in metrics

Harbor ships with:

- `sum`
- `min`
- `max`
- `mean`

## Custom metrics with `metric.py`

For package datasets, add custom aggregation via a `metric.py` file in `[[files]]`.

Create a dataset with a metric template:

```bash
harbor dataset init "my-org/my-dataset" --with-metric
```

The script receives:

- `-i, --input-path`: rewards JSONL file
- `-o, --output-path`: output JSON file with computed metrics

Example:

```python title="metric.py"
# /// script
# dependencies = []
# ///
import argparse
import json
from pathlib import Path


def main(input_path: Path, output_path: Path):
    rewards = []

    for line in input_path.read_text().splitlines():
        reward = json.loads(line)
        if reward is None:
            rewards.append(0)
        elif len(reward) != 1:
            raise ValueError(f"Expected one key per reward line, got {len(reward)}")
        else:
            rewards.extend(reward.values())

    output_path.write_text(json.dumps({"mean": sum(rewards) / len(rewards) if rewards else 0}))


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("-i", "--input-path", type=Path, required=True)
    parser.add_argument("-o", "--output-path", type=Path, required=True)
    args = parser.parse_args()
    main(args.input_path, args.output_path)
```

If you change `metric.py` and use non-local tasks, sync before publish:

```bash
harbor sync my-dataset
```

Then publish:

```bash
harbor publish my-dataset --public
```

`metric.py` output should be a JSON object. Multiple metric values are allowed:

```json
{"mean": 0.81, "pass_rate": 0.5}
```
