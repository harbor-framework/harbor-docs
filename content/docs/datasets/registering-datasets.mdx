Harbor datasets are `dataset.toml` manifests with task digests.

<Callout title="Legacy registry.json note">
You may still see older examples referring to `registry.json` and external PR-based registration.
Those are legacy patterns; the CLI package workflow below is the recommended approach for
creating and publishing datasets today.
</Callout>

## 1) Initialize a dataset manifest

Initialize the dataset manifest:

```bash
harbor dataset init "my-org/my-dataset" --description "A short description"
```

If you need a custom metric script:

```bash
harbor dataset init "my-org/my-dataset" --with-metric
```

This creates:

```
my-dataset/
├── dataset.toml
└── metric.py            # only if --with-metric
```

If you initialize in the same directory as `dataset.toml`, local tasks are auto-added when the manifest is generated (recommended, not required).
For example:

```bash
my-dataset/
├── dataset.toml
├── task-a/
├── task-b/
└── metric.py            # only if --with-metric
```

## 2) Add tasks and files

Use `harbor add` to explicitly add local, registered, or file-based dataset items.
Useful when tasks are not auto-added during manifest generation.

```bash
cd my-dataset
```

### Add local tasks

```bash
harbor add ./task-a ./task-b
```

### Add registered tasks or datasets

```bash
harbor add org/name-of-task --to my-dataset
harbor add org/name-of-dataset --to my-dataset
```

### Add all tasks from a local folder

```bash
harbor add ./candidate-tasks --scan --to my-dataset
```

### Optional metric file

If `dataset.toml` does not include `metric.py`:

```bash
harbor add metric.py --to my-dataset
```

`metric.py` must be added by filename and must be in the same directory as `dataset.toml`.

## 3) Sync task and file digests (when needed)

`harbor sync` is optional. It is for explicit digest refresh of local tasks in the dataset folder and registry-based task references.

Use it when you need to refresh remote task refs.

```bash
harbor sync my-dataset --upgrade
```

For local tasks inside the dataset folder, publish handles digest refresh automatically.
For local tasks outside the folder, re-add them to refresh digests; `harbor add` upgrades existing task entries by default if the task is already present.

## 4) Publish the dataset

Publish:

```bash
harbor publish my-dataset
```

```bash
harbor publish my-dataset --public -t v1.0 -t release-candidate
```

By default, publishing adds the `latest` tag to both tasks and datasets.
`harbor publish` refreshes local digests; use `harbor sync` for non-local/external refs.

For package-level dataset publishing with explicit behavior:

- `-t / --tag`: add one or more tags
- `--concurrency`: control parallel upload throughput
- `--no-tasks`: publish only dataset metadata/files
- `--public`: make package public (default is private)

For details on publishing workflow details, see [Publishing](/docs/datasets/publishing).

## 5) Validate your published dataset

After publishing, evaluate with:

```bash
harbor run -d "my-org/my-dataset@v1.0" -a "<agent>" -m "<model>"
```
